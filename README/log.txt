NEW
COMPONENT UPDATE: start.sh (formerly kayden.sh)
 
 SUMMARY:
 The system bootloader script has been finalized and standardized under the name start.sh. This script functions as the primary system orchestration layer responsible for bootstrapping, environment preparation, plugin registration, Git synchronization, kernel execution, and lifecycle management.
 
 The renaming from kayden.sh to start.sh is intended to clarify system entrypoint behavior while preserving internal identity branding within logs and runtime output.
 
 No changes were made to engine or kernel logic during this update.
 
 ⸻
 
 ==================================================
 CORE RESPONSIBILITIES OF start.sh
 
 1. SYSTEM BOOTSTRAP
 
 * Loads system environment implicitly via runtime context
 * Prepares execution environment for main.py kernel launch
 
 2. PLUGIN / TOOL / PATCH SCANNING (PnP SYSTEM)
 
 * Scans /tools and /patches directories
 * Builds active runtime manifests:
     * SVN_ACTIVE_TOOLS
     * SVN_ACTIVE_PATCHES
 * Exports manifests into environment for kernel access
 
 3. GIT REPOSITORY STATE MANAGEMENT
 
 * Fetches latest remote state
 * Detects repository status:
     * CLEAN
     * AHEAD
     * BEHIND
     * DIVERGED
 * Prompts user for sync decision with timeout fallback
 * Supports:
     * pull
     * push
     * skip (default on timeout)
 
 4. KERNEL INITIALIZATION
 
 * Launches FastAPI kernel via:
     * python3 main.py
 * Passes environment state into runtime system
 
 5. MEMORY AND STATE VAULTING
     After kernel shutdown or crash:
 
 * Commits persistent memory data:
     * identity/
     * state/
     * notes/
 * Pushes to main branch if changes detected
 
 6. EXPERIMENTAL CODE VAULTING
 
 * Switches to staging branch
 * Commits all code changes as snapshot
 * Pushes staging updates independently of main memory branch
 
 7. AUTO-RESTART LOOP
 
 * System runs in continuous cycle
 * On crash or exit:
     * logs event
     * performs vault operations
     * restarts after delay (5 seconds)
 
 ⸻
 
 ==================================================
 SYSTEM ROLE DEFINITION
 
 start.sh acts as:
 
 * BIOS / bootloader layer

 * Runtime environment injector
 * Kernel launcher (main.py)
 * Crash recovery loop manager
 * System lifecycle orchestrator
 
 It does NOT:

 * modify engine behavior
